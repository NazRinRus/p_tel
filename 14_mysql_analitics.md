### Аналитические функции в MySQL
#### Структура таблиц
```
CREATE TABLE stores (
    store_id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    address VARCHAR(50) NOT NULL
);

CREATE TABLE sales (
    sale_id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    store_id BIGINT REFERENCES stores (store_id),
    date TIMESTAMP NOT NULL,
    sale_amount DECIMAL(10,2) NOT NULL
);
```
#### Процедура генерации данных
`vim create_procedure_generate_test_data.sql`
```
DELIMITER $$

CREATE PROCEDURE generate_test_data()
BEGIN
    DECLARE days_range INT DEFAULT 730;

    SET @@cte_max_recursion_depth = 100000;
    
    START TRANSACTION;
    
    SET FOREIGN_KEY_CHECKS = 0;
    TRUNCATE TABLE sales;
    TRUNCATE TABLE stores;
    SET FOREIGN_KEY_CHECKS = 1;
    
    INSERT INTO stores (address) VALUES
    ('г. Москва, ул. Тверская, 1'),
    ('г. Москва, пр. Ленина, 15'),
    ('г. Санкт-Петербург, Невский, 30'),
    ('г. Казань, ул. Баумана, 5'),
    ('г. Екатеринбург, ул. Ленина, 20'),
    ('г. Новосибирск, Красный пр., 10'),
    ('г. Краснодар, ул. Красная, 50'),
    ('г. Нижний Новгород, ул. Большая, 8'),
    ('г. Ростов-на-Дону, пр. Ворошиловский, 12'),
    ('г. Самара, ул. Куйбышева, 25');
    
    INSERT INTO sales (store_id, date, sale_amount)
    WITH RECURSIVE numbers AS (
        SELECT 1 AS n
        UNION ALL
        SELECT n + 1 FROM numbers WHERE n < 100000
    )
    SELECT 
        IF(n <= 72000, 1, 2 + FLOOR(RAND() * 9)) AS store_id,
        DATE_SUB(NOW(), INTERVAL FLOOR(RAND() * days_range) DAY) AS date,
        ROUND(100 + RAND() * 49900, 2) AS sale_amount
    FROM numbers;
    
    COMMIT;
    
    SELECT 'Данные сгенерированы!' AS result, COUNT(*) AS total_sales FROM sales;
END$$

DELIMITER ;
```

#### Запросы
1. Запрос, который выведет нарастающий итог продаж по каждому магазину с группировкой по месяцам:
```
SELECT 
    s.store_id,
    st.address,
    DATE_FORMAT(s.date, '%Y-%m') AS month,
    SUM(s.sale_amount) AS monthly_sales,
    SUM(SUM(s.sale_amount)) OVER (
        PARTITION BY s.store_id 
        ORDER BY DATE_FORMAT(s.date, '%Y-%m')
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS cumulative_sales,
    COUNT(*) AS transactions_count
FROM sales s
JOIN stores st ON s.store_id = st.store_id
GROUP BY s.store_id, st.address, DATE_FORMAT(s.date, '%Y-%m')
ORDER BY s.store_id, month;
```

2. Запрос, который выведет 7-дневное скользящее среднее за последний месяц по самому плодовитому магазину:
```
WITH 
top_store AS (
    SELECT store_id
    FROM sales
    GROUP BY store_id
    ORDER BY COUNT(*) DESC
    LIMIT 1
),
daily_sales AS (
    SELECT 
        DATE(s.date) AS sale_date,
        SUM(s.sale_amount) AS daily_total,
        COUNT(*) AS transactions_count
    FROM sales s
    JOIN top_store ts ON s.store_id = ts.store_id
    WHERE s.date >= DATE_SUB(CURDATE(), INTERVAL 1 MONTH)
    GROUP BY DATE(s.date)
)
SELECT 
    sale_date,
    daily_total,
    transactions_count,
    ROUND(AVG(daily_total) OVER (
        ORDER BY sale_date 
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ), 2) AS moving_avg_7d,
    ROUND(MIN(daily_total) OVER (
        ORDER BY sale_date 
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ), 2) AS min_7d,
    ROUND(MAX(daily_total) OVER (
        ORDER BY sale_date 
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ), 2) AS max_7d
FROM daily_sales
ORDER BY sale_date;
```

3. Граничные случаи:
- Часовые пояса (Time Zones)
Риск: NOW() зависит от настройки сессии. Сервер может быть в UTC, а пользователь в Москве.
Решение: Хранить в UTC, конвертировать на уровне приложения или явно использовать CONVERT_TZ(NOW(), 'UTC', 'Europe/Moscow').
- Одновременный выбор «Топ-1» (Ties)
Риск: Если два магазина делят первое место, LIMIT 1 выбирает случайный.
Решение: Использовать оконную функцию RANK() вместо LIMIT, чтобы вывести всех лидеров.
