#### Запросы с соединениями:
- Соединение типа `LEFT JOIN`:
```
p_tel=# SELECT r.name, i.name FROM regions AS r LEFT JOIN institutions AS i ON r.id = i.region_id;
         name          |            name            
-----------------------+----------------------------
 Москва                | СИЗО-1
 Москва                | ИК-5 УФСИН
 Московская область    | 
 Санкт-Петербург       | Психиатрическая клиника №1
 Ленинградская область | 
 Свердловская область  | 
(6 строк)
```
При типе соединения `LEFT JOIN`, выводятся все строки левой таблицы и только те строки правой таблицы, которые нашли соответствие в левой. На месте строк правой таблицы, не нашедших соответствия, устанавливается значение `NULL`;
- Соединение типа `JOIN`:
```
p_tel=# SELECT r.name, i.name FROM regions AS r JOIN institutions AS i ON r.id = i.region_id;
      name       |            name            
-----------------+----------------------------
 Москва          | СИЗО-1
 Москва          | ИК-5 УФСИН
 Санкт-Петербург | Психиатрическая клиника №1
(3 строки)
```
Выводятся только те строки, которые нашли соответствие в левой и правой таблицах.

#### Запрос обновления данных используя UPDATE FROM
Установить статус всех терминалов учреждения в статус "MAINTENANCE", зная только название учреждения - "СИЗО-1"
```
p_tel=# UPDATE terminals t
SET status = 'MAINTENANCE', updated_at = CURRENT_TIMESTAMP
FROM institutions i
WHERE t.institution_id = i.id AND i.name = 'СИЗО-1';

#### Удаление данных с оператором DELETE используя join с другой таблицей с помощью using
Удалить терминалы, находящиеся в учреждениях со статусом `is_active = f`:
```
DELETE FROM terminals AS t
USING institutions AS i
WHERE t.institution_id = i.id AND i.is_active = 'f';
